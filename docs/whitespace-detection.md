# 빈영역(여백) 탐지 기술

> 카드 글씨 자동 배치에 사용하는 빈영역 탐지 기법 설명
> 구현: `src/lib/card-whitespace.ts` · API: `src/app/api/whitespace/route.ts`
> 작성일: 2026-06-11

## 개요

카드 배경 이미지에서 **텍스트가 들어갈 가장 비어있는(디테일 적은) 영역**을 찾는 기능.
**고전 컴퓨터비전 이미지 처리** 기술 조합으로, AI/ML이 아니라 수학적 연산이다.

- ⚡ 빠름: 분석 자체 ~20–40ms
- 💰 비용 0: 외부 API·모델 추론 없음
- 🎯 결정적(deterministic): 같은 이미지 → 항상 같은 결과

## 파이프라인 (5단계)

### 1. 다운스케일 + 그레이스케일 (sharp)
- 배경을 **96×144** 저해상도 흑백으로 축소
- 분석 속도 ↑, 디테일 노이즈 ↓

### 2. 라플라시안 엣지 검출 (Laplacian edge detection)
```
커널: [ -1  -1  -1
        -1   8  -1
        -1  -1  -1 ]
```
- 3×3 컨볼루션으로 경계/디테일이 많은 곳 = 높은 값, 평탄한 곳(여백) = 0에 가깝게 변환
- 결과 = "엣지 밀도 맵" (어디가 복잡하고 어디가 비었는지의 지도)

### 3. 적분영상 (Integral Image / Summed-Area Table)
- 누적합 테이블을 1회 생성 → **임의 사각형 영역의 합을 4번의 덧셈/뺄셈으로 O(1)** 계산
- 수천 번 영역을 평가해도 빠른 핵심 (Viola–Jones 얼굴검출에서 쓰인 기법)

```
windowSum = I[y1][x1] - I[y0][x1] - I[y1][x0] + I[y0][x0]
평균 밀도 = windowSum / (넓이) / 255   // 0~1, 낮을수록 비어있음
```

### 4. 씨앗 탐색 (Seed search)
- 작은 윈도우(가로 28% × 세로 10%)를 전체에 슬라이딩
- **엣지밀도가 가장 낮은(가장 빈) 핵심 지점**을 씨앗으로 선택

### 5. 영역 성장 (Region Growing)
- 씨앗에서 시작 → **인접 스트립의 평균 밀도가 임계값 이하인 동안 사방으로 확장**
- 임계값 `T = min(0.45, 씨앗밀도 + LOOSENESS)` (LOOSENESS 기본 0.07 = 느슨함)
- 과도 성장 방지 상한: 가로 ≤ 92%, 세로 ≤ 60%
- 성장 후 여유 패딩(2셀) 추가 → 흰 영역을 조금 더 포함
- 결과: 실제 흰 영역 모양에 맞게 박스가 **유동적으로** 커짐

## 반환값 (WhitespaceRegion)

| 필드 | 의미 |
|------|------|
| `x0,y0,x1,y1` | 탐지 박스 (정규화 0~1) |
| `cx,cy` | 박스 중심 (0~1) |
| `band` | `top` / `center` / `bottom` (cy 기준) |
| `density` | 평균 엣지밀도 0~1 (낮을수록 비어있음) |
| `emptiness` | `1 - density` (UI 표시용) |
| `wRatio,hRatio` | 박스 크기 비율 |

## 렌더링에서의 활용 (`card-text-render.ts`)

탐지 결과로 다음을 자동 결정:
- **밴드(상/중/하)**: 글씨 수직 배치 위치
- **자동 크기(Auto-fit)**: 영역 높이에 맞춰 제목/내용 폰트 크기 계산
- **적응형 글자색**: 영역 밝기(sharp `.stats()` 평균 휘도 < 135) → 어두우면 밝은 글자

## 파라미터 튜닝

| 상수 | 위치 | 기본값 | 효과 |
|------|------|--------|------|
| `AW`,`AH` | card-whitespace.ts | 96,144 | 분석 해상도 |
| `LOOSENESS` | card-whitespace.ts | 0.07 | 클수록 더 느슨/넓게 확장 |
| `MAX_W_RATIO` | card-whitespace.ts | 0.92 | 박스 최대 너비 |
| `MAX_H_RATIO` | card-whitespace.ts | 0.6 | 박스 최대 높이 |
| `PAD_CELLS` | card-whitespace.ts | 2 | 성장 후 여유 확장 |

API로 호출 시 `{ "bg": "<배경키/URL>", "looseness": 0.07 }` 형태로 `looseness` 조정 가능.

## 테스트

- 앱 내: 배경 선택 화면 → "🔍 빈영역 탐지 테스트" → "탐지 실행" → 박스 오버레이 + (band · 비어있음% · 크기% · ms)
- 스크립트: `node scripts/ws-detect.mjs <out-dir> <img...>` (초기 검증용, 고정크기 버전)

## 한계 / 향후 개선 후보

- 넓은 빈 영역이 없는 복잡한 사진은 차선책으로 피사체 위에 얹힐 수 있음 → 미리보기 수동 조정(배치/크기)으로 보완
- 정확도 향상 방향: 엣지 + 색 분산 결합, saliency 맵, 경량 객체검출 모델 등
